## 1. 프로젝트 개요

### 1.1 프로젝트명
AI 슬롭스쿼팅 탐지기 — LLM 패키지 환각 기반 공급망 공격 탐지 도구

### 1.2 문제 정의

LLM(대규모 언어 모델)은 코드 작성 시 실제로 존재하지 않는 패키지명을 추천하는 경우가 있다(환각, Hallucination). 공격자는 이를 악용하여 해당 이름으로 악성 패키지를 PyPI/npm에 선점 등록하고, AI 추천을 신뢰한 개발자가 이를 설치하면 악성코드가 실행된다. 이를 **슬롭스쿼팅(Slopsquatting)** 공격이라 한다.

### 1.3 연구 배경
- 슬롭스쿼팅 관련 논문 16편, 관련 데이터 576K+ 건 참조
- 실제 침해 사례: `react-codeshift` — 237개 리포지토리 영향
- PyPI/npm은 업로드 시 코드 리뷰 없이 즉시 등록 가능 → 공격자 등록부터 삭제까지의 시간 창이 공격 기회

### 1.4 프로젝트 목표 (제안서 기준)

| # | 목표 | 설명 |
|---|---|---|
| 1 | 실시간 존재 여부 검증 | PyPI/npm API를 통한 패키지 등록 상태 확인 |
| 2 | 위험도 기반 점수 산출 | 다중 시그널 기반 0~100점 위험도 판정 |
| 3 | 모델별 환각률 비교 분석 | GPT-4o, Claude, Gemini 등 LLM 간 환각 패턴 비교 |

## 3. 시스템 아키텍처

### 3.1 제안서 아키텍처 (단일 파이프라인)
```
LLM 모델 질문 → 패키지명 추출 → PyPI/npm 조회 → 위험도 점수 → HTML 리포트
```

### 3.2 현재 아키텍처 (3경로 + 2계층)

```
[경로 1: Chrome Extension — AI 사이트 실시간 감지]
AI 사이트 응답 → MutationObserver → 패키지명 추출
    → background.js (SHA-256 캐시) → FastAPI :8001
    → Layer 1 + Layer 2 → 인라인 결과 패널

[경로 2: CLI — 소스 파일 직접 분석]
소스 파일 → slop_check.py → FastAPI → Rich 터미널 출력

[경로 3: n8n — 자동화 워크플로우]
트리거 → Gemini API → 패키지 추출 → 분석 → HTML 리포트
```

### 3.3 기술 스택

| 계층 | 기술 |
|---|---|
| Chrome Extension | Manifest V3, content scripts (Claude/ChatGPT/Gemini) |
| API 서버 | FastAPI + uvicorn + httpx + rapidfuzz (Docker :8001) |
| 소스 분석 | 아카이브 메모리 스트리밍, 악성 패턴 9종, Shannon 엔트로피 |
| 코드 파서 | Python AST + Regex 하이브리드 (py/js/ts/json) |
| CLI | slop_check.py + Rich |
| 자동화 | n8n Workflow (:5679) |
| 연구 파이프라인 | OpenAI GPT-4o, SQLite, JSON/MD 리포트 |

## 4. 위험도 판정 체계

### 4.1 제안서 기준 (단일 계층)

| 시그널 | 점수 |
|---|:---:|
| PyPI/npm 미등록 | +55 |
| 7일 이내 신규 등록 | +25 |
| 유사 패키지 존재 | +20 |
| 합성 패턴 | +15 |
| 편집거리 (Levenshtein) | +5~20 |

### 4.2 현재 기준 (Layer 기반, 강화됨)

**Layer 1 — 메타데이터 분석 (모든 패키지)**

| 시그널 | 점수 | 변경 사항 |
|---|:---:|---|
| PyPI 미등록 | **+70** | 55 → 70 상향 |
| npm에도 미등록 | **+10** | 신규 추가 |
| 7일 이내 신규 등록 | +25 | 동일 |
| 30일 이내 등록 | +15 | 동일 |
| 합성 패턴 | +20 | 15 → 20 상향 |
| 편집거리 ≤2 | +20 | 동일 |
| 편집거리 ≤4 | +8 | 동일 |
| 버전 1개 | +10 | 동일 |
| npm에만 존재 (생태계 혼동) | +15 | 동일 |

**Layer 2 — 소스코드 분석 (등록된 패키지만)**

| 탐지 패턴 | 점수 | 설명 |
|---|:---:|---|
| install_hook | +20 | setup.py cmdclass 오버라이드 |
| exec_eval | +15 | exec(), eval(), compile() |
| base64_exec | +25 | base64 디코딩 + exec 조합 |
| credential_theft | +25 | os.environ + requests.post |
| shell_execution | +15 | subprocess, os.system |
| obfuscated_import | +10 | `__import__('os')` |
| network_access | +10 | socket, urllib 직접 사용 |
| npm_install_script | +25 | postinstall 악용 |
| high_entropy_string | +10 | Shannon 엔트로피 > 4.5 |

**신뢰도 할인 계수 (오탐 억제)**

등록된 패키지가 내부적으로 `eval()`, `subprocess` 등을 정상 목적으로 사용하는 경우의 오탐을 방지한다. 소스 점수에 신뢰도 할인 계수를 **곱하여** 최종 점수를 조정하며, 조건은 중첩 적용된다.

| 조건 | 할인 계수 |
|---|:---:|
| 등록 2000일 초과 (5년+) | × 0.2 |
| 등록 1000일 초과 (3년+) | × 0.3 |
| 등록 365일 초과 (1년+) | × 0.5 |
| 버전 50개 이상 | × 0.7 |
| 버전 10개 이상 | × 0.9 |
| 인기 패키지 목록 포함 (321개) | × 0.5 |

**의심 가산 (독립 시그널, 할인과 별도)**

| 조건 | 가산 |
|---|:---:|
| 등록 30일 이내 (신규) | +10 |
| 유사 이름 존재 (편집거리 ≤2) | +10 |
| 버전 1개 이하 | +5 |

```
최종 점수 = (소스 점수 × trust_factor) + suspicion
```

검증 예시:
```
pandas:   소스 40점 × 0.2(5953일) × 0.7(v114) × 0.5(인기) = 2점 → LOW
sklearn:  소스 10점 × 0.2(3925일) × 0.9(v10) + 10(유사)  = 11점 → LOW
신규 악성: 소스 60점 × 1.0(5일) + 10(신규) + 5(v1)       = 75점 → HIGH
미등록:    L1 메타데이터만 적용 → 80점                     = CRITICAL
```

**최종 판정 기준**

| 등급 | 점수 범위 | 의미 |
|---|:---:|---|
| CRITICAL | 80~100 | 즉시 위험 — 미등록이거나 악성 패턴 다수 |
| HIGH | 60~79 | 높은 위험 — 의심 시그널 복합 |
| MEDIUM | 30~59 | 주의 필요 — 일부 시그널 존재 |
| LOW | 0~29 | 안전 — 정상 등록 패키지 |

## 6. Chrome Extension 개발

### 6.1 제안서와의 관계
제안서 Future Roadmap에 "개발 환경(IDE) 실시간 연동"으로 언급되었던 기능을 **Chrome Extension** 형태로 구현하였다. IDE 대신 AI 사이트(Claude, ChatGPT, Gemini)를 직접 모니터링하여, 코드가 생성되는 시점에 개발자에게 즉시 경고한다.

### 6.2 지원 사이트 및 감지 방식

| 사이트 | 코드블록 감지 | 텍스트 감지 | 아티팩트 감지 |
|---|:---:|:---:|:---:|
| claude.ai | O | O | O (토큰 밀도 기반) |
| chatgpt.com | O | O | — |
| gemini.google.com | O | O (NLP 포함) | — |

### 6.3 감지 3단계

| 단계 | 방식 | 예시 | 신뢰도 |
|:---:|---|---|:---:|
| 1 | `pip install` / `npm install` 패턴 | `pip install flask numpy` | 높음 |
| 2 | 백틱 인라인 코드 + import 패턴 | `` `pandas` ``, `import torch` | 중간 |
| 3 | 130+ 인기 패키지 사전 매칭 | "Flask로 서버를 만들면..." | 중간 |

### 6.4 주요 버그 및 수정

| 버그 | 원인 | 수정 |
|---|---|---|
| 아티팩트 패널 여러 카드에 중복 | 스트리밍 중 코드 변화로 키가 매번 변경 | debounce + 카드 즉시 잠금 |
| 아티팩트 코드 미감지 | DOM 클래스명 하드코딩 (`min-w-0`) | 토큰 밀도 기반 동적 탐색 |
| iframe postMessage 패널 미생성 | 수신 핸들러가 로그만 출력 | 패널 생성 + 3단계 삽입 전략 |
| ChatGPT 무한 루프 | Set 제거 후 DOM 체크 실패 | `data-slop-scanned` 요소 마킹 |
| 동일 패키지 반복 시 UI 미표시 | Set 기반 중복 방지 | DOM 패널 존재 여부로 대체 |

## 9. 관련 프로젝트와의 차별점

유사 프로젝트인 **Phantom Guard**(github.com/matte1782/phantom_guard)와 비교하였다.

| 항목 | Phantom Guard | 본 프로젝트 |
|---|---|---|
| **탐지 시점** | 설치 전 (의존성 파일 스캔) | **AI 응답 실시간** (코드 생성 시점) |
| **소스코드 분석** | 없음 (메타데이터만) | **Layer 2: 아카이브 스트리밍 + 악성 패턴 9종** |
| **사용자 경로** | CLI / VS Code / GitHub Action | **Chrome Extension (3개 AI 사이트)** + CLI |
| **연구 모듈** | 없음 | **GPT-4o 환각률 측정 파이프라인** |
| **에코시스템** | PyPI + npm + Cargo | PyPI + npm |
| **CI/CD** | GitHub Action + pre-commit | n8n 워크플로우 |

**핵심 차별점:** 공격 타임라인에서 본 프로젝트는 ①번(AI가 코드 생성하는 시점)과 ⑤번(등록된 패키지의 소스 분석)을 커버하며, Phantom Guard는 ③번(의존성 파일 작성 후)을 커버한다. 상호 보완적이나, AI 사이트 실시간 감지와 소스코드 분석은 본 프로젝트만의 고유 기능이다.

## 13. 결론

제안서의 3가지 핵심 목표(실시간 검증, 위험도 판정, 모델 비교) 중 2가지를 완료하고 1가지를 부분 달성하였다. 제안서 범위를 넘어 **Layer 2 소스코드 분석**과 **Chrome Extension 실시간 감지**를 추가 구현하여, 단순한 메타데이터 검사 도구에서 **AI 코드 생성 시점의 공급망 보안 도구**로 발전시켰다.

남은 과제는 다중 모델 실험 확장과 Extension 안정화이며, 프로젝트 완료 시 AI 보조 개발 환경에서의 실질적인 보안 방어 체계를 제시할 수 있을 것으로 기대한다.
